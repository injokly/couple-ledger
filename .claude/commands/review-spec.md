# /review-spec

스펙을 구현하기 전에 검토한다. 모호한 부분, 누락된 요구사항, 다른 스펙과의 충돌을 찾는다.

사용법: `/review-spec specs/phase-1/03-transactions.md`

## 출력 형식

```
## 검토 결과: {스펙 경로}

### ✅ 명확한 부분
- ...

### ⚠️ 모호한 부분 (사용자 확인 필요)
1. [스펙 줄 N] "..." — 왜 모호한지
2. ...

### 🔗 다른 스펙과의 의존성
- 선행: ...
- 영향: ...

### 📋 구현 순서 제안
1. ...
2. ...

### 🚨 잠재 리스크
- ...
```

모호한 부분이 있으면 구현을 시작하지 않는다. 먼저 사용자 답을 받는다.
