# /implement-spec

스펙 파일 하나를 구현한다.

사용법: `/implement-spec specs/phase-1/03-transactions.md`

## 절차

1. 스펙 파일을 처음부터 끝까지 읽는다. 단 한 줄도 건너뛰지 않는다.
2. "개방 질문(Open Questions)" 섹션이 있으면 사용자에게 먼저 확인한다.
3. 의존성 섹션에 나열된 선행 작업이 완료되었는지 확인한다.
   - 안 됐으면 선행 작업을 먼저 하겠다고 제안한다.
4. 관련 기존 파일을 `grep`/`read`로 파악한다.
5. 구현 계획을 3~5줄로 요약해서 사용자에게 보여주고 승인받는다.
6. 구현한다. 순서: 타입 → API → 훅 → 컴포넌트 → 페이지 → 테스트.
7. 수용 기준 체크리스트를 하나씩 직접 검증하고 결과 보고한다.
8. 커밋 메시지를 `docs/conventions/commit-message.md` 포맷으로 제안한다.

## 금지

- 수용 기준을 "아마 통과할 것 같다"고 넘어가지 말 것
- 스펙에 없는 기능을 "이것도 있으면 좋을 것 같아서" 추가하지 말 것
- TypeScript `any` 사용 금지 (unknown 또는 구체 타입으로)
