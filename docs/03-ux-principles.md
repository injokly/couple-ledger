# 03. UX Principles

모든 화면/컴포넌트/기능이 이 원칙에 부합해야 한다. 충돌하면 이 문서를 따른다.

## 대원칙 1: 10초 규칙

**거래 1건 입력이 앱 실행 → 저장 완료까지 10초를 넘기면 안 된다.**

측정: 실제 폰에서 스톱워치로. 초 단위 재지 말고 느낌으로 넘기면 안 된다. 6~8초 목표, 10초는 최대치.

### 10초를 지키는 방법

1. **커스텀 숫자 패드 (시스템 키보드 금지)**
   - 시스템 키보드는 뜰 때 0.3~0.5초 지연 + 화면 절반 가림 + 숫자/문자 전환 마찰 발생
   - 모달 하단에 항상 노출되는 numpad 사용
2. **기본값 똑똑하게**
   - 날짜 = 오늘
   - 거래 타입 = 지출
   - 카테고리 = 최근 사용 상위 6~8개 먼저
   - 계좌 = 가장 최근 사용
3. **선택 단계 제거**
   - 메모는 선택 사항 (placeholder로 마찰 제거)
   - 태그 입력 UI는 기본 숨김
4. **낙관적 저장**
   - 저장 버튼 누르자마자 리스트에 반영, DB는 비동기
   - 실패 시 토스트 + 롤백

### 10초를 깨는 것들 (하지 말 것)

- 여러 단계 마법사 (step 1/2/3)
- 확인 대화상자
- 애니메이션 1초 이상
- 저장 버튼 누르고 로딩 인디케이터 표시
- 카테고리를 가나다 순으로 정렬 (빈도 순이어야 함)

## 대원칙 2: 부부 공유 모델

**모든 데이터는 household 단위 공유.** 개인 영역 없음.

- 둘 다 모든 거래 조회·생성·수정·삭제 가능
- 단, `created_by` 는 항상 기록 (누가 입력했는지 배지로 표시)
- 프라이버시 이슈 발생 시 `transactions.visibility` 컬럼 추가로 대응 (YAGNI, 지금 구현 금지)

## 대원칙 3: 5탭 + FAB 구조

```
[ 홈 ] [ 거래 ] [  + FAB  ] [ 자산 ] [ 설정 ]
```

- 하단 탭바 고정
- 중앙 FAB = 빠른입력 모달 트리거 (라우트 아님, 오버레이)
- 탭 5개 이상으로 늘리지 말 것
- 홈 아래 스크롤 안에 모든 핵심 정보 1-2 스크린 내 표시

## 대원칙 4: 3개의 리듬

화면은 사용 빈도에 최적화한다.

| 리듬 | 주 화면 | 최적화 기준 |
|---|---|---|
| **매일** | 빠른입력, 홈 요약 | 속도, 기본값, 낙관적 업데이트 |
| **매월** | 자산 스냅샷, 월간 리포트 | 정확성, 전월 비교, 재입력 편의 |
| **가끔** | 추이 차트, 목표, 설정 | 밀도, 탐색, 커스터마이즈 |

## 대원칙 5: 다크모드 Only (MVP)

- 다크모드가 유일한 테마
- 라이트모드 토글 만들지 말 것 (Phase 2+ 고려)
- 배경 #000000, 카드 #18191C
- 세부 토큰은 `docs/04-design-system.md` 참조

## 대원칙 6: 한국 주식 컨벤션

**상승 = 빨강 (#EB4D3D), 하락 = 파랑 (#3182F6)**

서구권 컨벤션(녹/적) 사용 금지. 사용자가 혼란스러워한다.

## 대원칙 7: 모바일 우선, 데스크톱은 선택

- 기본 뷰포트: 375px × 812px (iPhone 12 mini)
- 최대 컨테이너 너비: 420px (스마트폰 크기 유지)
- 데스크톱에서는 중앙 정렬된 "폰 화면"으로 표시 (Phase 1)
- Phase 2+ 에서 데스크톱 전용 레이아웃 고려

## 구체 UX 결정들

### 금액 표시 포맷
- 거래 리스트: `−12,500원` (1000단위 콤마, 원 단위까지)
- 홈 큰 숫자: `324,580,200원` (풀 표기)
- 요약 칩: `12.4M`, `1억 3천만원` (맥락에 맞게)
- 음수는 `−` (U+2212), 하이픈(`-`) 쓰지 말 것

### 날짜 표시
- 오늘: "오늘 12:48"
- 어제: "어제 19:34"
- 이번 주: "금요일 · 19:34"
- 이번 달: "4월 15일"
- 그 외: "2025년 4월 15일"

### 사용자 식별
- 컬러 도트 (6px)로 누가 입력한 거래인지 표시
- 아바타(32~36px)는 헤더에만, 리스트엔 도트로

### 카테고리 아이콘
- 이모지 사용 (MVP, Phase 2+ 에서 커스텀 아이콘 검토)
- 크기: 16~22px
- 배경 타일: 36×36, border-radius 11px, 카테고리별 dim 컬러

## 접근성

- 터치 타겟 최소 44×44
- 컬러만으로 의미 구분 금지 (항상 +/− 기호 병행)
- Focus 링 항상 표시 (outline: 2px solid var(--accent))
- 스크린리더 레이블 (한국어)
