# 00. Overview — 우리 가계

## 누가 쓰는가

부부 2명. 둘 다 소유자 권한, 데이터 완전 공유. 강인조(남편, 은행 PL, 기술 능숙) + 임채원(배우자, 네이버 ProductManger, 기획 능숙)

## 무엇을 해결하는가

기존 가계부 앱의 3가지 문제:

1. **입력이 번거롭다** — 매번 카테고리 선택, 계좌 선택, 금액 입력에 30초+
2. **자산 전체가 안 보인다** — 현금만 보거나, 주식만 보거나. 부동산·대출·투자가 흩어져 있음
3. **"그래서 몇 년 뒤 뭐가 되는데"에 답이 없다** — 과거 기록은 많은데 미래 시뮬레이션이 없다

## 무엇을 만드는가

> 월말에 한 번 스냅샷만 찍어도 순자산 추이가 보이고,
> 매일 점심값 기록이 10초 안에 끝나며,
> 몇 년 뒤 자산이 얼마가 될지 차트로 볼 수 있는 앱.

## 핵심 기능 우선순위

### Must Have (Phase 1)

- 수입/지출/이체 거래 기록
- 계좌·자산 카테고리 관리
- **월말 자산 스냅샷** (순자산 추이의 원천)
- 홈 대시보드 (순자산, 현금흐름, 자산 구성)
- 거래 내역 조회 & 편집

### Should Have (Phase 2)

- 반복 거래 자동 생성
- 주식·펀드 시세 연동
- 카테고리별 예산 & 초과 알림
- 월간 가계 리포트 (규칙 기반)
- 목표 기반 복리 시뮬레이션

### Could Have (Phase 3)

- AI 리밸런싱 제안 (Claude API)
- 거시 지표 연동 (환율, 금리, 지수)
- 다음 달 지출 예측

### Won't Have (이번 범위 외)

- 오픈뱅킹 직접 연동 (수동 입력 + CSV 업로드로 우회)
- 영수증 OCR
- 소셜 기능, 비교/랭킹
- 법인/사업자 기능

## 사용 리듬

세 가지 접속 빈도를 기억:

| 빈도 | 상황                 | 주 화면                     |
| ---- | -------------------- | --------------------------- |
| 매일 | 지출 기록            | 빠른입력 모달               |
| 매월 | 수입·자산 정리       | 자산 스냅샷 폼, 월간 리포트 |
| 가끔 | 추이 확인, 목표 점검 | 홈 차트, 자산 탭            |

화면 설계는 이 빈도에 맞춰야 한다.

## 성공 지표

- 부부 둘 다 30일 연속 입력 (MVP 출시 후)
- 평균 거래 입력 시간 < 10초
- 월말 자산 스냅샷 작성률 > 90%
- 6개월 후에도 유지되는 습관화
